# Video Autopsy — YouTube 영상 제작방식 역분석 에이전트

## 개요

YouTube 영상의 제작 방식을 역분석하여 상세 리포트를 생성하는 스킬이다.
사용자가 YouTube URL을 제공하면, 영상을 다운로드하고 키프레임과 오디오를 추출한 뒤,
Claude의 Vision 능력으로 각 프레임을 직접 분석하여 종합 리포트를 출력한다.

**핵심 원칙**: 별도 API 호출 없이 Claude Code의 내장 Vision 능력만으로 분석한다.
따라서 추가 비용이 발생하지 않는다.

## 사전 요구사항

아래 도구가 시스템에 설치되어 있어야 한다:

- **yt-dlp**: 영상 다운로드
- **ffmpeg**: 키프레임/오디오 추출

설치 확인 방법:
```bash
yt-dlp --version
ffmpeg -version
```

만약 미설치 상태라면:
```bash
# Windows (PowerShell)
winget install yt-dlp
winget install ffmpeg

# macOS
brew install yt-dlp ffmpeg

# Linux
sudo apt install ffmpeg && pip install yt-dlp
```

## 분석 워크플로우

사용자가 YouTube URL을 제공하면, 아래 단계를 **순서대로** 실행한다.

### Step 1: 작업 디렉토리 준비

프로젝트 루트에 임시 작업 폴더를 만든다.

```bash
mkdir -p ./video-autopsy-temp
cd ./video-autopsy-temp
```

### Step 2: 영상 메타데이터 수집

먼저 영상의 기본 정보를 수집한다. 이 정보는 리포트 상단에 표시된다.

```bash
yt-dlp --dump-json --no-download "VIDEO_URL" > metadata.json
```

metadata.json에서 추출할 항목:
- `title`: 영상 제목
- `channel`: 채널명
- `channel_follower_count`: 구독자 수
- `view_count`: 조회수
- `duration`: 영상 길이 (초)
- `upload_date`: 업로드일
- `width`, `height`: 해상도
- `description`: 영상 설명

### Step 3: 영상 다운로드

최대 720p로 다운로드한다 (분석 목적이므로 고해상도 불필요).

```bash
yt-dlp -f "bestvideo[height<=720]+bestaudio/best[height<=720]" \
  -o "video.%(ext)s" \
  --merge-output-format mp4 \
  "VIDEO_URL"
```

### Step 4: 키프레임 추출

영상 길이에 따라 추출 간격을 조정한다:
- 60초 이하 (Shorts): 2초 간격 → 최대 30장
- 60초~300초: 5초 간격 → 최대 60장
- 300초 이상: 10초 간격 → 최대 30장

최종 분석에는 **10~15장**을 선별하여 사용한다.

```bash
# 예: 3초 간격 추출
ffmpeg -i video.mp4 -vf "fps=1/3" -q:v 2 frame_%04d.jpg
```

추출된 프레임 중 대표성 있는 프레임을 선별하는 기준:
- 첫 프레임 (인트로 확인)
- 마지막 프레임 (아웃트로 확인)
- 장면 전환이 일어나는 지점
- 중간 균등 배분

### Step 5: 오디오 추출

```bash
ffmpeg -i video.mp4 -vn -acodec pcm_s16le -ar 16000 audio.wav
```

오디오 파형 정보 확인:
```bash
ffmpeg -i audio.wav -af "volumedetect" -f null /dev/null 2>&1
```

### Step 6: 장면 전환 감지

프레임 간 변화량을 측정하여 장면 전환 빈도를 파악한다.

```bash
ffmpeg -i video.mp4 -filter:v "select='gt(scene,0.3)',showinfo" -vsync vfr -f null /dev/null 2>&1 | grep "showinfo"
```

이 결과에서:
- 총 장면 전환 횟수
- 평균 장면 전환 간격 (초)
- 전환 패턴 (규칙적 vs 불규칙)

## 분석 항목 (4개 레이어)

추출한 프레임과 오디오 정보를 바탕으로 아래 4개 레이어를 분석한다.

### 🎨 레이어 1: 비주얼 분석

추출된 키프레임 이미지들을 **직접 보고** 아래 항목을 판별한다.

| 항목 | 판별 기준 |
|------|-----------|
| **영상 유형** | 슬라이드형 / 이미지 나열형 / 모션그래픽형 / 실사편집형 / 혼합형 |
| **이미지 소스** | AI생성 / 스톡이미지 / 직접촬영 / 스크린캡처 / 일러스트 |
| **AI 생성 여부** | Midjourney풍 (과도한 디테일, 특유의 광택) / DALL-E풍 (약간 평면적) / Stable Diffusion풍 (다양한 스타일) / Flux풍 (포토리얼) / Sora풍(AI동영상) |
| **아트스타일** | 카툰 / 수채화 / 3D렌더링 / 픽셀아트 / 포토리얼 / 애니메이션풍 / 일러스트 |
| **색감/톤** | 밝고 화사한 / 어둡고 무거운 / 파스텔 / 고채도 / 저채도 |
| **텍스트/자막** | 유무, 위치(상단/하단/중앙), 폰트스타일, 언어 |
| **워터마크** | 편집도구 워터마크 유무 (CapCut, Canva 등) |
| **캐릭터 일관성** | 동일 캐릭터가 반복 등장하는지 (시드 고정 추정) |

**AI 생성 이미지 판별 포인트**:
- 손가락 개수/형태 이상
- 텍스트 렌더링 오류
- 배경의 불자연스러운 반복 패턴
- 과도한 광택/하이라이트
- 피부 질감의 지나친 매끄러움
- 눈동자의 비대칭 또는 이상한 반사

### 🔊 레이어 2: 오디오 분석

오디오 파형과 메타데이터를 기반으로 분석한다.

| 항목 | 판별 기준 |
|------|-----------|
| **음성 유형** | TTS / 사람목소리 / 없음 |
| **TTS 추정** | 균일한 피치, 자연스럽지 않은 호흡, 일정한 속도 → TTS 가능성 높음 |
| **TTS 엔진 추정** | ElevenLabs(매우 자연스러움) / Google TTS(기계적) / Clova(한국어) / TYPECAST(한국어, 감정표현) |
| **BGM** | 유무, 분위기(밝음/잔잔/긴박/일렉트로닉) |
| **효과음** | 유무, 빈도, 종류(팝/띠링/전환음) |
| **음성 언어** | 한국어/영어/일본어/중국어 등 |

**TTS 판별 보조 기준**:
- 완벽히 균일한 볼륨 → TTS 가능성 높음
- 호흡 소리 없음 → TTS 가능성 높음  
- 미세한 감정 변화 없음 → TTS 가능성 높음
- 발음이 지나치게 정확 → TTS 가능성 높음

### ✂️ 레이어 3: 편집 패턴 분석

장면 전환 데이터와 프레임 비교를 통해 분석한다.

| 항목 | 판별 기준 |
|------|-----------|
| **장면 전환 빈도** | 평균 N초마다 전환 (Step 6 결과 활용) |
| **전환 효과** | 하드컷 / 페이드 / 줌인아웃 / 슬라이드 |
| **Ken Burns 효과** | 정지 이미지에 줌/팬 모션이 적용되었는지 |
| **텍스트 오버레이** | 화면에 텍스트가 오버레이되는 패턴과 빈도 |
| **영상 구조** | 인트로-본문-아웃트로 비율 |
| **반복 패턴** | 특정 장면/효과가 주기적으로 반복되는지 |
| **썸네일 vs 본문** | 썸네일과 본문 비주얼의 일관성 |

### 🛠️ 레이어 4: 제작 도구 추정

위 3개 레이어의 분석 결과를 종합하여 제작 파이프라인을 추정한다.

**추정 로직**:

1. **스크립트/기획**: 
   - 구조화된 나레이션 → ChatGPT / Claude 등 AI 활용 추정
   - 개인적 경험담 → 직접 작성 추정

2. **이미지 생성**:
   - AI 생성 → Midjourney / DALL-E / Stable Diffusion / Flux
   - 스톡 이미지 → Shutterstock / Unsplash / Pexels
   - 직접 제작 → Photoshop / Illustrator / Procreate

3. **음성**:
   - TTS → ElevenLabs / Google TTS / Clova / TYPECAST
   - 직접 녹음 → 마이크 + 오디오 편집

4. **편집/조합**:
   - 워터마크 기반 → CapCut / Canva
   - 전환효과 스타일 → Premiere Pro / Final Cut / DaVinci
   - 간단한 슬라이드 → CapCut / Canva / Vrew
   - AI 영상 → Sora / Runway / Pika

5. **자막**:
   - 자동 자막 스타일 → Vrew / CapCut 자동자막
   - 디자인 자막 → After Effects / Premiere Pro

## 정리 작업

분석 완료 후 임시 파일을 정리한다. 단, 리포트 파일은 보존한다.

```bash
# 영상 파일과 프레임 삭제 (용량이 크므로)
rm -f ./video-autopsy-temp/video.*
rm -f ./video-autopsy-temp/frame_*.jpg

# metadata.json과 report.md는 보존
```

사용자에게 리포트 파일 경로를 안내하고, 원하면 삭제할지 물어본다.

## 주의사항

- 영상 다운로드는 분석 목적 임시 사용이며, 분석 후 반드시 삭제한다.
- 제작 도구 추정은 100% 정확하지 않을 수 있으며, 신뢰도를 함께 표기한다.
- Shorts(60초 이하)와 일반 영상의 분석 전략이 다르므로 영상 길이를 먼저 확인한다.
- 프레임 추출 시 저장 공간을 고려하여 적절한 간격과 장수를 설정한다.
- 오디오 분석에서 TTS 판별은 추정이므로 "추정"이라는 표현을 반드시 사용한다.
