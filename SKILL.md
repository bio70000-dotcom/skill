# 트렌드 리서처 스킬 — 떡상 발굴 + 영상 역분석 + 쿼터 관리

## 역할 요약

Paperclip 조직 내 **트렌드 리서처 에이전트**가 사용하는 스킬.
두 가지 핵심 기능을 통합한다:

1. **떡상 영상 발굴** — Hot Video Finder v3 로직(VSR, 등급, 채널 검증)
2. **영상 제작방식 역분석** — Video Autopsy (S등급 영상을 해부하여 "어떻게 만들었는지" 파악)

> **핵심 원칙: 영상 주제가 최우선.**
> 모든 주제는 "쇼츠 또는 롱폼 영상으로 만들 수 있는가"가 1차 판단 기준이다.
> 블로그(neworbit.co.kr) 연동은 영상 주제가 확정된 후 부가적으로 검토한다.

# Phase 1: 떡상 영상 발굴

## ⓪ 쿼터 관리 (가장 먼저 실행)

### YouTube Data API v3 쿼터 체계

```
일일 쿼터 총량: 10,000 units (매일 태평양시간 자정에 리셋)

API 호출별 소모량:
  search.list       → 100 units (가장 비쌈!)
  videos.list       → 3 units (영상 50개 배치 가능)
  channels.list     → 5 units (채널 50개 배치 가능)
  playlistItems.list → 3 units
```

### 1회 검색의 쿼터 소모 시뮬레이션

```
키워드 1개 검색 시:
  search(인기순 50개)        = 100 units
  search(최신7일 30개)       = 100 units
  videos.list(~80개 배치2회) = 6 units
  channels.list(~40개)       = 5 units
  playlistItems(~40채널)     = 120 units (3 × 40)
  videos.list(최근5×40채널)  = 120 units (3 × 40)
  ─────────────────────────────────────
  소계: ~451 units / 키워드
```

### 쿼터 배분 전략

```
┌───────────────────────────────────────────────────┐
│  일일 쿼터 배분 계획                                │
├───────────────────────────────────────────────────┤
│  오전 검색 (메인)           5,000 units (50%)      │
│    → 핵심 키워드 5~6개 × ~450 units                │
│                                                    │
│  Video Autopsy              1,000 units (10%)      │
│    → S등급 1~2개 영상 역분석 (API 불필요, 시간만)    │
│                                                    │
│  오후 보충 검색 (선택)      2,000 units (20%)      │
│    → 오전 S등급 관련 추가 검색 또는 시즌 키워드       │
│                                                    │
│  여유분 (예비)              2,000 units (20%)      │
│    → 긴급 트렌드 / 다른 에이전트 사용                │
└───────────────────────────────────────────────────┘
```

> **참고**: Video Autopsy는 YouTube Data API를 쓰지 않는다 (yt-dlp + ffmpeg).
> 따라서 쿼터를 소모하지 않지만, 다운로드/분석에 시간이 걸린다.

### 쿼터 절약 팁

1. **채널 중복 제거**: 같은 채널 여러 키워드 → 최근5 검증은 1회만
2. **이전 기록 활용**: 어제 검색한 카테고리는 오늘 스킵
3. **search 대신 playlistItems**: 검증된 채널은 3u로 모니터링 (100u 절약)
4. **short 필터**: 쇼츠용이면 `duration=short`로 필터링
5. **max_results 조절**: 첫 검색 30개, S등급 나오면 50개로 확장

## ③ VSR 계산 + 등급 산정

### VSR = 영상 조회수 ÷ 채널 구독자 수

### 구독자 규모별 기대 VSR

```
0 ~ 1,000           → 기대 0.80
1,000 ~ 10,000      → 기대 0.40
10,000 ~ 100,000    → 기대 0.20
100,000 ~ 1,000,000 → 기대 0.10
1,000,000+          → 기대 0.04
```

### 배수 = 실제 VSR ÷ 기대 VSR

### 등급

```
S+: 10배+  |  S: 6~10배  |  A: 3~6배
B+: 2~3배  |  B: 1.2~2배 |  C: 0.7~1.2배  |  D: <0.7배
```

**주제 후보: A등급 이상(배수 3배+)만 통과. S등급 이상은 Video Autopsy 대상.**

## ④ 채널 최근 5개 영상 검증

### 일관성 (CV)
```
CV < 0.3 → 매우 안정 | < 0.6 → 안정 | < 1.0 → 보통 | >= 1.0 → 불안정
```

### 추세 (최신2개 vs 과거 평균)
```
>= 1.5 → 급상승📈 | >= 1.1 → 상승↑ | >= 0.9 → 유지→ | >= 0.6 → 하락↓ | < 0.6 → 급하락📉
```

### 종합 판정
```
🔥 진짜 떡상 채널    (기대×5 + 안정)     → ★★★ + Autopsy 대상
💥 떡상 진행 중      (기대×3 + 상승)     → ★★★ + Autopsy 대상
💪 실력파           (기대×3 + 안정)     → ★★☆
⚡ 폭발력 있음      (기대×3 + 불안정)   → ★★☆
👍 기대 이상        (기대×1.5)          → ★☆☆
➡️ 평균 / 📉 미만   (나머지)            → ❌ 탈락
```

## ⑤ 시의성 + 적합도

### 시의성 (1~10)

```
9~10: 48시간 이내 (바이럴, 긴급)
7~8:  이번 주 (시즌 이벤트)
5~6:  2주 이내 (트렌드)
3~4:  한 달 (계절, 기념일)
1~2:  에버그린 (상식, 심리, 과학)
```

### 시의성 → 영상 유형 매핑

```
시의성 9~10 → 쇼츠 즉시 제작 (48초, 빠른 소비)
시의성 7~8  → 쇼츠 우선 + 여유 있으면 롱폼도
시의성 5~6  → 쇼츠 또는 롱폼 선택
시의성 3~4  → 롱폼 위주
시의성 1~2  → 롱폼 에버그린
```

### 적합도 (1~5) — 영상 제작 가능성이 핵심

```
5: 완벽 — AI 파이프라인(Gemini→Grok→TTS→FFmpeg)으로 바로 제작 가능
4: 좋음 — 약간의 리서치/각도 조정 후 제작 가능
3: 보통 — 제작은 되지만 차별화 고민 필요
2: 약함 — 실사/리액션 등 우리 파이프라인에 안 맞음
1: 부적합 — 제작 불가

체크:
  ✅ 정보 전달, 상식, 심리, 스토리텔링, 밈 재해석 → 4~5점
  ✅ 이거알아 스타일 8컷 구성에 맞는 주제 → 5점
  ⚠️ 비교/리뷰형 (실물 필요할 수 있음) → 2~3점
  ❌ 실시간 리액션, 얼굴 필수, 실사 촬영 → 1점
```

# Phase 2: Video Autopsy — S등급 영상 제작방식 역분석

Phase 1에서 **S등급 이상 + 채널 검증 통과**한 영상 중 1순위 1~2개에 대해
제작 방식을 역분석한다. 대본작가에게 "무슨 주제"뿐 아니라 "어떻게 만들었는지"까지 전달하기 위함이다.

## ⑥ Video Autopsy 실행

### 사전 요구사항

```bash
# 설치 확인
yt-dlp --version
ffmpeg -version

# 미설치 시 (Windows PowerShell)
winget install yt-dlp
winget install ffmpeg
```

### Step 1: 작업 디렉토리 + 메타데이터 수집

```bash
mkdir -p ./video-autopsy-temp
yt-dlp --dump-json --no-download "VIDEO_URL" > ./video-autopsy-temp/metadata.json
```

추출할 항목: title, channel, channel_follower_count, view_count, duration, upload_date, width, height

### Step 2: 영상 다운로드 (720p)

```bash
yt-dlp -f "bestvideo[height<=720]+bestaudio/best[height<=720]" \
  -o "./video-autopsy-temp/video.%(ext)s" \
  --merge-output-format mp4 "VIDEO_URL"
```

### Step 3: 키프레임 추출

영상 길이에 따라 간격 조정:
```
60초 이하 (Shorts): 2초 간격 → 최대 30장
60~300초: 5초 간격 → 최대 60장
300초 이상: 10초 간격 → 최대 30장
```

최종 분석에는 **10~15장을 선별**한다 (첫/마지막 프레임 + 장면 전환점 + 균등 배분).

```bash
ffmpeg -i video.mp4 -vf "fps=1/3" -q:v 2 frame_%04d.jpg
```

### Step 4: 오디오 추출 + 파형 분석

```bash
ffmpeg -i video.mp4 -vn -acodec pcm_s16le -ar 16000 audio.wav
ffmpeg -i audio.wav -af "volumedetect" -f null /dev/null 2>&1
```

### Step 5: 장면 전환 감지

```bash
ffmpeg -i video.mp4 -filter:v "select='gt(scene,0.3)',showinfo" -vsync vfr -f null /dev/null 2>&1 | grep "showinfo"
```

결과에서: 총 전환 횟수, 평균 전환 간격(초), 패턴(규칙적 vs 불규칙)

### Autopsy에서 뽑아내야 할 핵심 인사이트 (대본작가에게 전달)

1. **우리 파이프라인으로 재현 가능한가?** → 적합도 점수 보정

## 주의사항

1. **쿼터 최우선**: 검색 전 잔여 쿼터 확인. 1,000u 미만이면 검색 중단
2. **Autopsy는 S등급만**: 모든 영상을 역분석하면 시간 낭비. 1순위 1~2개만
3. **Autopsy 영상은 분석 후 삭제**: 다운로드는 임시, 리포트만 보존
4. **이전 기록 활용**: 매일 새로 검색하지 말고 기록 누적으로 패턴 파악
5. **S등급 이상만 추천**: B+ 이하 탈락
6. **원히트 원더 필터링**: 최근5 검증 필수
7. **에버그린 30%**: 시의성 높은 영상 70% + 에버그린 30%
8. **블랙리스트**: 3연속 D등급 키워드는 2주간 스킵
9. **영상이 본업**: 블로그 연동은 부가. 영상 적합성 항상 우선
10. **Autopsy 리포트 영구 보관**: 벤치마킹 자산으로 누적
