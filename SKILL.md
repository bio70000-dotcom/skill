# 트렌드 리서처 스킬 — 떡상 영상 발굴 + 쿼터 관리 + 기록 누적

## 역할 요약

Paperclip 조직 내 **트렌드 리서처 에이전트**가 사용하는 스킬.
YouTube Data API v3 **일일 10,000 units 쿼터** 안에서 최대한 효율적으로
떡상 영상을 발굴하고, 이전 기록을 누적 분석하여 주간 주제 리스트를 제출한다.

> **핵심 원칙: 영상 주제가 최우선.**
> 모든 주제는 "쇼츠 또는 롱폼 영상으로 만들 수 있는가"가 1차 판단 기준이다.
> 블로그(neworbit.co.kr) 연동은 영상 주제가 확정된 후 부가적으로 검토한다.


## ⓪ 쿼터 관리 (가장 먼저 실행)

### YouTube Data API v3 쿼터 체계

```
일일 쿼터 총량: 10,000 units (매일 태평양시간 자정에 리셋)

API 호출별 소모량:
  search.list       → 100 units (가장 비쌈!)
  videos.list       → 3 units (영상 50개 배치 가능)
  channels.list     → 5 units (채널 50개 배치 가능)
  playlistItems.list → 3 units
```

### 1회 검색의 쿼터 소모 시뮬레이션

```
키워드 1개 검색 시:
  search(인기순 50개)        = 100 units
  search(최신7일 30개)       = 100 units
  videos.list(~80개 배치2회) = 6 units
  channels.list(~40개)       = 5 units
  playlistItems(~40채널)     = 120 units (3 × 40)
  videos.list(최근5×40채널)  = 120 units (3 × 40)
  ─────────────────────────────────────
  소계: ~451 units / 키워드

키워드 5개 검색 시: ~2,255 units
키워드 10개 검색 시: ~4,510 units (하루의 거의 절반!)
```

### 쿼터 배분 전략

**하루 10,000 units를 아래처럼 배분한다:**

```
┌─────────────────────────────────────────────┐
│  일일 쿼터 배분 계획                          │
├─────────────────────────────────────────────┤
│  오전 검색 (메인)         6,000 units (60%)  │
│    → 핵심 키워드 5~6개 × ~450 units          │
│                                              │
│  오후 보충 검색 (선택)    2,000 units (20%)  │
│    → 오전 S등급 나온 주제 관련 추가 검색       │
│    → 또는 시즌 키워드 2~3개                   │
│                                              │
│  여유분 (예비)            2,000 units (20%)  │
│    → 긴급 트렌드 발생 시 수동 검색용           │
│    → 또는 다른 에이전트(퍼블리셔 등)가 사용    │
└─────────────────────────────────────────────┘
```

### 쿼터 절약 팁

1. **채널 중복 제거**: 같은 채널이 여러 키워드에서 나오면 최근5 검증은 1회만
   - app.py의 `seen_channels` 캐시가 이미 이걸 한다
2. **이전 기록 활용**: 어제/그제 이미 검색한 카테고리는 오늘 스킵
3. **search 대신 videos**: 특정 채널 모니터링은 search(100u) 대신 playlistItems(3u)로
4. **short 영상 필터**: 쇼츠용이면 `duration=short`로 필터링하여 불필요한 영상 제거
5. **max_results 조절**: 첫 검색은 30개로 시작, S등급 나오면 해당 키워드만 50개로 확장

### 쿼터 추적

각 검색 후 app.py가 `metadata.api_calls_used`에 소모량을 기록한다.
이 값을 누적해서 오늘 남은 쿼터를 추적한다.

```python
# 오늘 사용한 쿼터 계산 (data/ 폴더의 오늘 날짜 파일들)
today_files = [f for f in data_dir.glob("*.json")
               if f.name.startswith(today_str)]
used = sum(json.load(f)["metadata"]["api_calls_used"]
           for f in today_files)
remaining = 10000 - used
```


## ③ VSR 계산 + 등급 산정

### VSR = 영상 조회수 ÷ 채널 구독자 수

### 구독자 규모별 기대 VSR

```
0 ~ 1,000           → 기대 0.80
1,000 ~ 10,000      → 기대 0.40
10,000 ~ 100,000    → 기대 0.20
100,000 ~ 1,000,000 → 기대 0.10
1,000,000+          → 기대 0.04
```

### 배수 = 실제 VSR ÷ 기대 VSR

### 등급

```
S+: 10배+  |  S: 6~10배  |  A: 3~6배
B+: 2~3배  |  B: 1.2~2배 |  C: 0.7~1.2배  |  D: <0.7배
```

**주제 후보 기준: A등급 이상 (배수 3배+) 만 통과**

## ④ 채널 최근 5개 영상 검증

### 일관성 (CV)
```
CV < 0.3 → 매우 안정 | < 0.6 → 안정 | < 1.0 → 보통 | >= 1.0 → 불안정
```

### 추세 (최신2개 vs 과거 평균)
```
>= 1.5 → 급상승📈 | >= 1.1 → 상승↑ | >= 0.9 → 유지→ | >= 0.6 → 하락↓ | < 0.6 → 급하락📉
```

### 종합 판정
```
🔥 진짜 떡상 채널    (기대×5 + 안정)     → ★★★
💥 떡상 진행 중      (기대×3 + 상승)     → ★★★
💪 실력파           (기대×3 + 안정)     → ★★☆
⚡ 폭발력 있음      (기대×3 + 불안정)   → ★★☆
👍 기대 이상        (기대×1.5)          → ★☆☆
➡️ 평균 / 📉 미만   (나머지)            → ❌ 탈락
```

## ⑤ 시의성 + 적합도

### 시의성 (1~10)

```
9~10: 48시간 이내 (바이럴, 긴급)
7~8:  이번 주 (시즌 이벤트)
5~6:  2주 이내 (트렌드)
3~4:  한 달 (계절, 기념일)
1~2:  에버그린 (상식, 심리, 과학)
```

### 시의성 → 영상 유형 매핑 (영상이 본업!)

```
시의성 9~10 → 쇼츠 즉시 제작 (48초, 빠른 소비)
시의성 7~8  → 쇼츠 우선 + 여유 있으면 롱폼도
시의성 5~6  → 쇼츠 또는 롱폼 선택
시의성 3~4  → 롱폼 위주 (깊이 있는 콘텐츠)
시의성 1~2  → 롱폼 에버그린 (블로그 연동은 부가로만)
```

> **원칙**: 모든 주제는 "영상으로 만들 수 있는가"가 1차 기준.
> 블로그 연동은 영상 주제가 확정된 후 "이거 글로도 쓸 수 있겠다" 수준으로만 메모.

### 적합도 (1~5) — 영상 제작 가능성이 핵심

```
5: 완벽 — AI 파이프라인(Gemini→Grok→TTS→FFmpeg)으로 바로 제작 가능
4: 좋음 — 약간의 리서치/각도 조정 후 제작 가능
3: 보통 — 제작은 되지만 차별화 고민 필요
2: 약함 — 실사/리액션 등 우리 파이프라인에 안 맞음
1: 부적합 — 제작 불가

체크:
  ✅ 정보 전달, 상식, 심리, 스토리텔링, 밈 재해석 → 4~5점
  ✅ 이거알아 스타일 8컷 구성에 맞는 주제 → 5점
  ⚠️ 비교/리뷰형 (실물 필요할 수 있음) → 2~3점
  ❌ 실시간 리액션, 얼굴 필수, 실사 촬영 → 1점
```

## ⑦ 이전 기록 교차 분석 (핵심 차별화)

쿼터를 절약하면서도 트렌드를 정확히 잡으려면,
**매일의 데이터를 누적해서 패턴을 읽는 것**이 가장 중요하다.

### A. 반복 S등급 주제 탐지

```
최근 7일 기록에서 같은 주제(또는 유사 키워드)가
3회 이상 S등급으로 나타나면 → "확정 핫 트렌드"

예:
  3/28: "심리 테스트" 키워드 → S등급 영상 3개
  3/29: "MBTI" 키워드 → S등급 영상 2개
  3/31: "성격 유형" 키워드 → S등급 영상 4개
  → 결론: "심리/성격" 카테고리가 확정 핫 트렌드!
```

### B. 떡상 채널 추적

```
이전 기록에서 🔥/💥 판정 받은 채널 목록을 추적:
  - 이 채널의 신규 업로드를 우선 감시 (playlistItems만 3 units!)
  - search(100 units)보다 30배 이상 저렴
  - 이미 검증된 채널이므로 최근5 재검증 불필요
```

### C. 실패 키워드 블랙리스트

```
최근 3회 연속 D등급만 나온 키워드 → 블랙리스트
  → 해당 키워드에 쿼터 낭비하지 않음
  → 2주 후 재시도 (트렌드 변화 가능성)
```

### D. 쿼터 사용 패턴 분석

```
최근 7일의 일별 쿼터 사용량:
  월: 8,200 units → 오버페이스 ⚠️
  화: 5,100 units → 적절 ✅
  수: 3,800 units → 여유 있음
  목: 6,500 units → 적절
  금: 2,100 units → 너무 적음 (놓친 트렌드 있을 수 있음)

주간 평균: 5,140 units/일
권장: 4,500~6,000 units/일 범위 유지
```

### E. 주간 트렌드 리포트 (매주 월요일)

이전 기록 7일치를 종합하여 주간 리포트를 생성한다:

```markdown
## 주간 트렌드 리포트 (3/24 ~ 3/30)

### 이번 주 핫 카테고리 (S등급 빈도 순)
1. 심리/성격 — S등급 12개 (3일 연속 출현)
2. 동물/펫 — S등급 8개 (꾸준)
3. IT/테크 — S등급 6개 (상승 추세)

### 이번 주 발굴된 떡상 채널
| 채널 | 판정 | 첫 발견일 | S등급 영상 수 |
|------|------|----------|-------------|
| 채널A | 🔥 진짜 떡상 | 3/25 | 4개 |
| 채널B | 💥 떡상 진행 중 | 3/27 | 3개 |

### 쿼터 사용 현황
- 주간 총 사용: 35,980 / 70,000 units (51.4%)
- 일 평균: 5,140 units
- 가장 효율적이었던 검색: "심리 상식" (451u → S등급 5개)
- 가장 비효율적이었던 검색: "뷰티 메이크업" (451u → D등급만)

### 다음 주 검색 계획
- 우선 키워드: 심리/성격 확장, IT 신제품
- 스킵 키워드: 뷰티 (3연속 D등급)
```

## ⑧ 주간 주제 리스트 제출

### 최종 점수

```
최종 = (등급배수 × 0.25) + (채널판정 × 0.2) + (시의성 × 0.2) + (적합도 × 0.25) + (반복출현 × 0.1) + 보너스

보너스:
  시리즈화 가능 +0.5 | 쇼츠 최적 주제 +0.5
  댓글 유도력 높음 +0.3 | 동일 주제 S등급 3일+ 연속 +1.0
  이전 기록에서 같은 채널 재등장 +0.5
  블로그 연동도 가능 +0.2 (부가 보너스, 필수 아님)
```

## 주의사항

1. **쿼터 최우선**: 검색 전 반드시 잔여 쿼터 확인. 1,000u 미만이면 검색 중단
2. **이전 기록 활용**: 매일 새로 전부 검색하지 말고, 기록 누적으로 패턴 파악
3. **채널 모니터링은 저렴하게**: 검증된 채널은 playlistItems(3u)로, search(100u) 금지
4. **S등급 이상만 추천**: B+ 이하 탈락
5. **원히트 원더 필터링**: 최근5 검증 필수
6. **에버그린 30%**: 시의성 높은 영상 주제 70% + 에버그린 30% (영상이 본업, 블로그는 부가)
7. **블랙리스트 관리**: 3연속 D등급 키워드는 2주간 스킵
8. **블로그는 부가**: neworbit.co.kr 연동은 있으면 좋지만, 영상 주제 적합성이 항상 우선
